# Peter의 AI 코딩 10가지 원칙  
> *How to code without writing code…*

AI 에이전트와 함께 일하면서 생산성을 극대화하기 위한 실전 원칙 정리입니다.  
핵심 메시지는 **“코드 집착을 버리고, 시스템과 흐름에 집중하라”** 입니다.

---

## 1. 완벽주의를 버리세요 (Kill Perfectionism)

- 내 스타일과 100% 일치하지 않아도 OK
- 동작하면 일단 넘어가는 유연함이 필요
- 완벽함보다 **속도와 반복**이 중요
- 이 태도가 AI와 협업할 때 가장 큰 자산이 됨

---

## 2. 에이전트가 스스로 검증하게 설계하세요 (Close the Loop)

- 컴파일 → 린트 → 실행 → 테스트 → 검증까지 에이전트가 직접 수행
- 사람이 중간에 계속 확인하는 구조는 병목이 됨
- 피드백 루프를 최대한 짧게 유지해야 반복 속도가 빨라짐

---

## 3. Pull Request는 죽었고, Prompt Request 시대입니다 (PR is Dead)

- 코드보다 **그 코드를 만든 프롬프트**가 더 중요
- 외부 PR은 최소화, 핵심 아이디어만 프롬프트로 재활용
- 팀 내부에서도 “프롬프트 자산화”가 중요해짐

---

## 4. 코드 리뷰 대신 아키텍처 토론만 하세요 (Discuss Architecture Only)

- 구현 디테일은 에이전트의 몫
- 시스템 구조, 큰 결정, 방향성만 논의
- Discord 등에서도 코드 이야기는 하지 않음
- 팀 전체에 이 관점이 정착되는 것이 중요

---

## 5. 여러 에이전트를 병렬로 돌리세요 (Parallel Agents)

- 하나의 작업에 오래 매달리지 않기
- 여러 작업을 병렬로 큐잉
- 계획 → 에이전트에 위임 → 다음 에이전트로 이동
- 이렇게 해야 **몰입 상태(Flow State)** 유지 가능

---

## 6. 계획 단계에 과할 정도로 시간을 쓰세요 (Plan Hard)

- 실행 전, 에이전트와 충분히 핑퐁하며 계획을 다듬음
- 도전 → 수정 → 반박 → 만족할 때까지 반복
- 계획이 탄탄하면 실행 단계 개입이 거의 필요 없음
- Codex 선호 이유: 실행 중 질문이 적어 흐름이 끊기지 않음

---

## 7. 일부러 모호하게 지시하세요 (Be Vague)

- 너무 구체적이면 AI가 그 범위 안에서만 움직임
- 의도적으로 여지를 남겨 예상 못 한 해결책을 유도
- 실제로 해보면 효과가 큼 (항상은 아니지만 매우 유용)

---

## 8. 원격 CI 기다리지 말고 로컬에서 바로 테스트하세요 (Local First)

- 원격 CI 대기 10분은 낭비
- 에이전트가 로컬에서 직접 테스트하도록 설계
- 피드백 루프를 짧게 유지해야 반복 속도가 빨라짐

---

## 9. 대부분의 코드는 지루한 데이터 변환입니다 (Just Data X-Form)

- 앱 코드 대부분은  
  “데이터를 이 형태에서 저 형태로 바꾸는 일”
- 이런 작업에 집착할 필요 없음 → 에이전트에게 위임
- 에너지는 코드가 아니라 **시스템 설계**에 써야 함

---

## 10. 문제 해결자보다 제품 출시형 인간이 AI에 유리합니다 (Builders > Solvers)

- 알고리즘 퍼즐 좋아하는 개발자가 오히려 AI 전환에 불리
- 구현 디테일보다는 **결과물과 출시**에 관심 있는 사람이 빠르게 적응
- 핵심은 **Ship it!**

---

## Peter의 미래 전망 (보너스)

- 수많은 앱은 사라지고 API만 남을 것
- 예:
  - MyFitnessPal을 직접 열 필요 없이
  - 음식 사진만 에이전트에게 보내면
  - 칼로리 계산 + 건강 목표 수정까지 자동 처리

---

## 정리 한 줄 요약

> **완벽주의를 버리고, 코드 리뷰 대신 아키텍처를 논의하며,  
> 에이전트가 스스로 검증하게 만들고, 여러 에이전트를 병렬로 돌려라.**

AI 시대의 개발자는  
**“코드를 잘 짜는 사람”이 아니라  
“시스템을 잘 설계하고, 빠르게 출시하는 사람”** 입니다.
