주요 기능

📝

일기 작성 및 저장

로컬 데이터베이스(SQLite)에 안전하게 저장되며, 날짜별로 일기를 관리하고 조회할 수 있습니다.

🤖

AI 감정 분석

Groq API (Llama 3.3)를 활용하여 실시간에 가까운 감정 분석을 제공합니다. 캐릭터 선택에 따라 공감 톤이 달라집니다.

📊

감정 통계 대시보드

감정 추이 차트, 감정 정원(🌱→🌻), 키워드 태그 클라우드로 나의 감정 패턴을 시각화합니다.

💬

맞춤형 추천

AI가 분석한 감정 상태에 기반하여 공감 메시지와 맞춤형 행동 지침을 제안합니다.

🔒

프라이버시 보호

모든 일기 데이터는 로컬 기기에만 저장되어 개인 정보가 안전하게 보호됩니다.

📱

반응형 UI

320dp~600dp+ 다양한 화면 크기에 최적화된 반응형 디자인으로 모든 기기에서 편안하게 사용할 수 있습니다.

최근 개선

v1.4.40에서는 FCM 푸시 알림의 안정성을 근본적으로 개선했습니다. Data-only payload 전환으로 중복 알림을 해결하고, 3-Layer 빈 알림 방어 체계를 구축했으며, 실시간 알림 진단 UI를 추가하여 사용자가 직접 알림 상태를 확인할 수 있습니다.

Architecture

FCM Data-Only Payload 전환

Android 백그라운드/killed 상태에서 OS 자동 표시 + 핸들러 이중 실행으로 발생하던 중복 알림 문제를 근본적으로 해결했습니다. 서버 FCM 메시지를 notification → data-only로 전환하고, iOS는 APNS alert로 분리 처리합니다. 고정 notification ID(2001)로 덮어쓰기 안전장치도 추가했습니다.

Reliability

3-Layer 빈 알림 방어 체계

Background Isolate에서 NotificationService 미초기화로 빈 알림이 표시되던 문제를 해결했습니다. buildPersonalizedMessage → handler fallback → showNotification guard의 3단계 방어로, 어떤 상황에서도 의미 있는 메시지가 표시됩니다. Background Isolate 초기화 로직도 추가했습니다.

UX

알림 진단 대시보드

NotificationDiagnosticService를 신규 구축하여 알림 예약 상태, 정확한 알람 권한, 배터리 최적화, 시간대 정보를 실시간으로 수집합니다. 설정 화면에 진단 위젯을 추가해 사용자가 알림 상태를 한눈에 확인하고, 문제 시 원인을 즉시 파악할 수 있습니다.

Quality

Analytics 강화 + 테스트 7개 추가

알림 스케줄링 시 scheduleMode(exact/inexact)와 timezone 정보를 Analytics에 기록하여 디버깅 추적성을 높였습니다. FCM 빈 문자열 방어 테스트 4개, showNotification 빈 알림 방어 테스트 3개를 추가하여 알림 시스템의 신뢰성을 검증합니다.
