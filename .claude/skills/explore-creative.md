# explore-creative

창의적 탐색 스킬 (`/explore-creative [goal]`)

Peter의 AI 코딩 원칙 7: "일부러 모호하게 지시하세요"

## 목표
- 열린 질문으로 다양한 해결책 탐색
- 에이전트의 창의성을 활용한 혁신적 접근
- 구체적 지시 대신 목표 중심 탐색

## 트리거 조건
다음 상황에서 이 스킬을 실행합니다:
- `/explore-creative [목표]` 명령어
- "창의적으로 해결해줘" 요청
- 새로운 접근법이 필요한 상황
- 기존 방법이 효과적이지 않을 때

## 핵심 철학

### 구체적 지시 vs 모호한 지시
```
❌ 구체적: "ListView.builder를 사용해서 일기 목록을 구현해줘"
✅ 모호함: "일기 목록을 보여주는 가장 좋은 방법을 찾아줘"

❌ 구체적: "Provider를 invalidate해서 데이터를 새로고침해줘"
✅ 모호함: "데이터 동기화 문제를 해결해줘"

❌ 구체적: "버튼을 왼쪽으로 옮겨줘"
✅ 모호함: "사용자가 더 쉽게 접근할 수 있게 해줘"
```

## 프로세스

### Step 1: 목표 해석
```
입력: /explore-creative "사용자 경험 개선"

1. 목표의 범위 파악
2. 암묵적 제약 조건 식별
3. 성공 기준 정의 (열린 형태)
```

### Step 2: 다중 관점 탐색 (병렬)
```
3개 에이전트가 서로 다른 관점에서 접근:

Agent 1 (Innovator): 혁신적 접근
  - 기존 패턴 무시
  - 새로운 방법론 탐색
  - "만약 처음부터 다시 만든다면?"

Agent 2 (Optimizer): 최적화 접근
  - 기존 코드 개선점 식별
  - 점진적 개선 방안
  - "현재 상태에서 가장 효과적인 변화는?"

Agent 3 (User-Focused): 사용자 관점
  - UX 중심 분석
  - 사용자 pain point 식별
  - "사용자가 진짜 원하는 것은?"
```

### Step 3: 아이디어 수집
각 에이전트가 2-3개 아이디어 제시:
```
Innovator:
  1. 음성 기반 일기 입력
  2. AI가 질문하고 사용자가 답하는 인터뷰 형식
  3. 사진에서 감정 자동 추출

Optimizer:
  1. 일기 작성 시간 50% 단축
  2. 자동 저장 + 드래프트
  3. 템플릿 시스템

User-Focused:
  1. 감정 입력 간소화 (1-tap)
  2. 리마인더 개선
  3. 통계 시각화 강화
```

### Step 4: 아이디어 평가
사용자에게 옵션 제시:
```
┌─────────────────────────────────────────────────────────┐
│                   창의적 탐색 결과                      │
├─────────────────────────────────────────────────────────┤
│ 목표: 사용자 경험 개선                                  │
│                                                         │
│ 발견된 접근법:                                          │
│                                                         │
│ [A] 혁신적 변화                                         │
│     • 음성 일기 입력 도입                              │
│     • 예상 공수: 높음 / 영향: 매우 높음               │
│                                                         │
│ [B] 점진적 개선                                         │
│     • 1-tap 감정 입력 + 자동 저장                      │
│     • 예상 공수: 낮음 / 영향: 중간                    │
│                                                         │
│ [C] 퀵 윈                                               │
│     • 통계 시각화 강화                                 │
│     • 예상 공수: 중간 / 영향: 높음                    │
│                                                         │
│ 어떤 방향으로 진행할까요?                              │
└─────────────────────────────────────────────────────────┘
```

### Step 5: 선택된 방향 구체화
사용자 선택 후 구체적 실행 계획 수립

```
═══════════════════════════════════════════════════════════
              Creative Exploration: 사용자 경험 개선
═══════════════════════════════════════════════════════════

Phase: 다중 관점 탐색 중

Agent Status:
├── [>] Innovator        탐색 중... (기존 패턴 분석)
├── [>] Optimizer        탐색 중... (병목 지점 식별)
└── [>] User-Focused     탐색 중... (UX 분석)

현재: 각 에이전트가 독립적으로 아이디어 발굴
```

```
발견된 아이디어: 9개
실현 가능 후보: 5개

   - 구현 복잡도: 높음

2. [사용자] 1-tap 감정 입력

   - 구현 복잡도: 낮음

3. [최적화] 통계 시각화 강화

다음 단계
├── 위 옵션 중 선택

```

## 사용 예시

### 성능 개선 탐색
```
> "/explore-creative 앱이 더 빠르게 느껴지도록"

AI 응답:
1. 3개 관점 병렬 탐색
   - Innovator: 선형 로딩 → 스켈레톤 UI
   - Optimizer: 이미지 지연 로딩, 캐싱
   - User-Focused: 체감 속도 개선 (애니메이션)
2. 5개 후보 아이디어 제시
3. 사용자 선택 대기
```

### 기능 아이디어
```
> "/explore-creative 사용자 참여도를 높이는 방법"

AI 응답:
1. 3개 관점 병렬 탐색
2. 발견:
   - 일일 리마인더 개선
   - 연속 기록 스트릭
   - 친구와 감정 공유 (익명)
3. 영향도/복잡도 매트릭스 제시
```

## 모호함의 효과

| 구체적 지시 | 모호한 지시 |
|-------------|-------------|
| 예상 가능한 결과 | 놀라운 아이디어 발견 가능 |
| 빠른 실행 | 탐색에 시간 투자 |
| 안전함 | 혁신 가능성 |
| 작은 범위 | 넓은 범위 |

**사용 시점**:
- 새로운 기능 아이디어가 필요할 때
- 기존 방법이 효과적이지 않을 때
- 사용자 요구가 불명확할 때
- 혁신이 필요할 때

## 연관 스킬
- `/parallel-dev` - 병렬 개발 (구체적 실행)
- `/feature-pipeline` - 기능 개발 파이프라인
- `/swarm-review` - 다관점 리뷰

## 주의사항
- 긴급한 버그 수정에는 부적합 (구체적 지시 필요)
- 명확한 요구사항이 있을 때는 오버헤드
- 결과의 불확실성 존재
- 탐색 후 반드시 사용자 선택 단계 필요
